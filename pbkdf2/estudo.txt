Password-Based Key Derivation Function 2 - PBKDF2

OBJETIVO:
 Criar um hash e atrasar propositalmente o processamento para frustrar
 ataques de força bruta ou dicionário.

ARQUITETURA:
 Entradas necessárias:
 - Senha (P) -> texto original do usuário.
 - Salt (S) -> valor aleatório (evita ataques com tabelas pré-computadas).
 - Iterações (c) -> número de repetições (ex: 100.000+).
 - Função hash (H) -> geralmente SHA-256.
 - Tamanho da chave desejada (dkLen) -> ex: 256 bits.
 - Motor de hash (PRF): a função pseudoaleatória base (HMAC-SHA256)

 Fluxo de processamento:
  1. Primeira passada (U1): o HMAC é acionado usando Senha como chave.
   A mensagem processada é o Salt concatenado com um número de bloco
   (iniciando em 1). A saída é um bloco de 32 bytes (capacidade do SHA256).

  2. O Loop de Iterações (U2 até Uc): a saída exata do passo anterior (u1)
   entra novamente no HMAC como mensagem, sempre utilizando Password como
   chave. Isso se repete em cadeia pelo número exato de iterações (c).
 
  3. A Combinação (XOR): todos os resultados intermediários gerados
   (U1, U2, ..., Uc) são misturados bit a bit usando a operação lógica
   matemática XOR (^). O resultado único deste imenso XOR é o "Bloco 1"
   válido.
 
  4. Extensão (Concatenação): o SHA256 gera blocos de 32 bytes. Se você
   pedir uma chave final (dkLen) de 64 bytes, o algoritmo repete os passos
   1 a 3 inteiros, mas agora alterando o número do bloco inicial para 2
   (Salt + 2). No fim, ele junta (concatena) o Bloco 1 com o Bloco 2 para
   entregar os 64 bytes.

RESUMO:
 O PBKDF2 atrasa propositalmente o processamento ao:
 - Misturar Senha + Salt;
 - Aplicar o HASH;
 - Repetir milhares de vezes;
 - Cominar os resultados com XOR.

POR QUE MÚLTIPLAS ITERAÇÕES?
- Sem iterações o atacante testa milhões de senhas por segundo;
- Com PBKDF2 cada tentativa fica 100.000x mais lenta, como exemplo.

PAPEL DO SALT:
- Garantir que quenhas iguais gerem chaves diferentes.
- Impedir Rainbow Tables.
- Deve ser:
  - Aleatório;
  - Único por usuário;
  - Armazenado junto com o hash.

ATIVIDADE:
 1. Desenvolver um método que implemente o AES
 2. O método deve receber um texto e uma senha e retornar o conteúdo
  criptografado.
